# Contributing to EnvatoBirthday
Thank you for considering contributing to EnvatoBirthday. We appreciate your interest in helping us improve this project.

## Getting Started
To contribute to EnvatoBirthday, follow these steps:

1. **Fork the repository**: Create a fork of the EnvatoBirthday repository by clicking the "Fork" button in the top-right corner of the repository page. This will create a copy of the repository in your own GitHub account.
2. **Clone the repository**: Clone your fork of the repository to your local machine using the command `git clone https://github.com/your-username/EnvatoBirthday.git`.
3. **Create a new branch**: Create a new branch for your contribution using the command `git checkout -b feature/your-feature-name`. For more information on branch naming, see the [Branch Naming](#branch-naming) section.

## Branch Naming
When creating a new branch, use the following naming conventions:

* `feature/` for new features
* `fix/` for bug fixes
* `docs/` for documentation changes
* `chore/` for miscellaneous changes

For example: `feature/add-new-birthday-theme`

## Commit Messages
We use [Conventional Commits](https://www.conventionalcommits.org/) to format our commit messages. This means that your commit messages should follow the format:

`type(scope): brief description`

Where:

* `type` is one of:
	+ `feat` for new features
	+ `fix` for bug fixes
	+ `docs` for documentation changes
	+ `chore` for miscellaneous changes
* `scope` is optional and should be used to specify the area of the project that the commit affects
* `brief description` should be a brief summary of the changes made in the commit

For example: `feat(birthday-themes): add new theme options`

## Pull Request Checklist
Before submitting a pull request, make sure to:

* [ ] Run `git status` to ensure that all changes are committed
* [ ] Run `git diff --staged` to review the changes to be committed
* [ ] Ensure that all commit messages follow the [Conventional Commits](https://www.conventionalcommits.org/) format
* [ ] Ensure that all changes are relevant to the branch and do not introduce any new bugs
* [ ] Test the changes thoroughly to ensure that they work as expected
* [ ] Update the documentation if necessary
* [ ] Include a brief description of the changes made in the pull request

By following these guidelines, you can help us improve the quality and consistency of the EnvatoBirthday project. Thank you for your contribution!
